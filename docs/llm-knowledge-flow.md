# LLM Knowledge Flow Integration

This project implements the executable part of a local-first Joplin knowledge
pipeline. The `gatelynch/llm-knowledge-base` framework is partially adopted as
a workflow layer and directory model, not as a package dependency.

## Local Mapping

| gatelynch layer | This project | Notes |
| --- | --- | --- |
| `raw/` | `notes_root/` | Joplin SQLite export target and source collection input. With notebook filtering enabled, files live under `notes_root/<joined-notebook-slug>/<safe-title>.md`. Treat exported files as read-only evidence. |
| `wiki/` | `wiki_root/` | Compiled wiki generated by `wiki-compile` or `agent-compile`; notebook-scoped output lives under `wiki_root/<joined-notebook-slug>/` and is indexed into the wiki vector collection. |
| `brainstorming/` | `brainstorming/` | Human/LLM exploration logs and health reports. |
| `artifacts/` | `artifacts/` | Finished work products. Keep manually authored output here before deciding whether to import it back into Joplin or another source flow. |

## Initialized Directories

```text
notes_root/                 # existing source/raw layer; gitignored
  <notebook-slug>/          # e.g. 工作-專案A-會議/
    <safe-title>.md
wiki_root/                  # existing compiled wiki layer; gitignored
  <notebook-slug>/
    index.md
    topics/
  summaries/                # compatibility target for source-level summaries
  concepts/                 # compatibility target for cross-source concepts
  indexes/                  # compatibility target for All-Sources / All-Concepts
brainstorming/
  chat/
  health/
artifacts/
  projects/
```

## Operating Rules

- Language: 繁體中文.
- Knowledge-management outputs should use Traditional Chinese for human-readable
  content. Keep technical proper nouns, source paths, and filenames in their
  original language when clearer.
- Prefer these compiled-wiki section headings when relevant: 核心結論, 關鍵證據,
  疑點, 術語, 我的主張, 實踐經驗, 我的實踐, 外部觀點, 張力與缺口.
- Do not edit existing files under `notes_root/` by hand; they are Joplin export output when `joplin_sqlite_sync.reconcile_mode: mirror` is enabled.
- Notebook-filtered exports use joined notebook slugs. For a nested notebook
  path such as `工作/專案A/會議`, write source files under
  `notes_root/工作-專案A-會議/` and compile wiki files under
  `wiki_root/工作-專案A-會議/`.
- Source filenames are title-based. Stable tracking belongs in frontmatter:
  `joplin_note_id`, `joplin_notebook_id`, `joplin_notebook_path`, and
  `joplin_notebook_slug`.
- Use `pnpm exec joplin-llm-wiki sqlite-sync --config ./config.yaml --export-only` to refresh the raw/source layer from Joplin.
- Use `pnpm exec joplin-llm-wiki sqlite-sync --config ./config.yaml --select-notebooks` to update notebook selection in `config.yaml` without exporting immediately.
- Use `pnpm exec joplin-llm-wiki index --config ./config.yaml` to index `notes_root/` and `wiki_root/`.
- Use `pnpm exec joplin-llm-wiki wiki-compile --config ./config.yaml` for local Ollama compilation into `wiki_root/`.
- Use `pnpm exec joplin-llm-wiki agent-compile --config ./config.yaml` for Codex CLI agent compilation. This uses local `codex exec`, not an OpenAI API key.
- Store exploratory Q&A, thinking notes, and compile/health observations in `brainstorming/chat/` or `brainstorming/health/`.
- Store completed outputs in `artifacts/`; when an artifact should affect future compilation, import it into Joplin or another configured source path instead of placing it inside `notes_root/` directly.

## Model Selection

| Mode | Command | Model boundary |
| --- | --- | --- |
| Local model | `wiki-compile` | Uses Ollama for planner/writer and the configured Ollama embedding model for indexing. This is the default local path. |
| Codex Agent | `agent-compile` | Uses local `codex exec` with an already logged-in ChatGPT/Codex subscription. It does not use OpenAI API billing and should emit Traditional Chinese wiki output. |
| OpenAI API | N/A | Not implemented in this repo. Do not assume a ChatGPT/Codex monthly subscription grants API access. |

## Current Difference From Upstream

The upstream workflow compiles both `raw/` and `artifacts/` into `wiki/`.
This project currently compiles the configured `notes_root/` source tree and
the existing `wiki_root/` tree. Because `notes_root/` is managed by Joplin
SQLite mirroring and notebook filters, `artifacts/` is intentionally kept
outside that tree during initialization to avoid accidental deletion or source
pollution.
